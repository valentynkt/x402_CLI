## 3. Philosophy & Values

### Public Good Orientation

**Principle:** Build infrastructure that enables ecosystems, not products that compete within them.

**Manifestation:**
- Prefer creating the "rails" over the "train"
- Infrastructure-first thinking
- Enable other builders to succeed
- Open source by default
- Community ownership over company control

**Example:** Would rather build the authentication layer that every Solana AI agent uses than build a single AI agent application.

### B2B / Developer-Facing Preference

**Principle:** Tools for builders, not tools for consumers.

**Preference Hierarchy:**
1. **Most Preferred:** Developer tools, SDKs, infrastructure, APIs
2. **Acceptable:** B2B SaaS tools, platform components
3. **Least Preferred:** B2C consumer applications, user-facing products

**Reasoning:**
- Deeper technical challenges
- Higher leverage (developers build on your work)
- Better alignment with expertise
- More enjoyable work

### Enablement Over Competition

**Principle:** Fill gaps in the ecosystem, don't compete with existing ecosystem tools.

**Application to x402 Ecosystem:**
- **Do NOT compete with:** Corbits/Faremeter, Coinbase CDP SDK, PayAI Network, x402scan
- **DO enable:** Integration layers, testing infrastructure, identity systems, authentication
- **DO complement:** Provide missing pieces that make existing tools better
- **DO collaborate:** Build what the ecosystem needs but no single company will build

**Strategic Positioning:** "I make your tool more secure/useful/accessible" rather than "Use my tool instead of theirs"

### Open Source Commitment

**Principle:** Fully open, community-owned solutions.

**License Preference:** MIT, Apache 2.0, or GPL (depending on use case)

**Why This Matters:**
- Aligns with Solana Foundation's public goods focus
- Builds trust and adoption
- Enables community contributions
- Demonstrates genuine ecosystem-first thinking

## 11. Decision-Making Framework

### When Choosing What to Build

**Ask These Questions:**

1. **Gap Test:** Does this fill a genuine gap in the x402/Solana AI ecosystem?
   - If duplicates existing tool → Reject
   - If enables existing tools → Consider

2. **Expertise Test:** Does this showcase my core capabilities?
   - MCP servers ✅
   - OAuth/identity ✅
   - Backend engineering ✅
   - Developer tooling ✅
   - Frontend/design ❌

3. **Achievability Test:** Can I complete this in 7-8 days working solo?
   - Scope is focused and clear ✅
   - Requires extensive team collaboration ❌
   - Heavy frontend work ❌

4. **Alignment Test:** Does this align with Solana Foundation priorities?
   - Agent identity/authentication ✅
   - Developer tooling ✅
   - Security infrastructure ✅
   - Public goods ✅
   - Consumer apps ❌

5. **Values Test:** Is this a public good that enables the ecosystem?
   - Enables other tools ✅
   - Competes with existing tools ❌
   - Open source ✅
   - Proprietary/closed ❌

6. **Prize Test:** Does this target 2+ prize tracks?
   - Single niche track ⚠️
   - Multiple compatible tracks ✅

**Decision Rule:**
- 5-6 ✅ marks → Strong candidate
- 3-4 ✅ marks → Consider with modifications
- 0-2 ✅ marks → Reject
